React的核心概念

发表于 2020-04-21

继续学习React的核心概念。

JSX

什么是JSX

  • 由 Facebook 起草的 js 扩展语法
  • 本质是一个JS对象，会被Babel编译，最终会被转化为creatElement
  • 每一个JSX表达式，有且仅有一个根节点（可以是React.Fragment）
  • 每个JSX元素必须结束

在JSX中嵌入表达式

  • {} 中写js表达式，相当于 ${}

    • null, undefined, false 不会显示
    • 子元素不可以放普通对象，可以放React元素对象
    • 可以放数组，循环渲染要加key值

  • 将表达式作为元素属性，className={red}

  • 属性使用小驼峰命名, className毕竟是js代码

  • 防止注入式攻击

    • 自动编码
    • dangerouslySetInnerHTML

React元素不可变

Read Only -> Object.freeze(obj)

  • JSX元素是一个对象，这个对象的属性不可更改
  • 每一次变化需要重新渲染（重新创建js对象，React看哪里变化了改哪里）

注意的点

  • 工程中图片不能写相对路径，要引入图片

React组件和属性

组件：包含内容、样式、功能的 UI 单元

创建组件

组件的名称首字母必须大写

  1. 函数组件 React Function Component(rfc)
    要返回一个React元素(JSX、React.createElement)
    使用组件，生成的是一个React元素，变化的是type值为f 函数组件

  2. 类组件 React Class Component(rcc)
    必须继承React.Component
    必须提供render函数，用于渲染组件
    使用类组件，生成的是一个React元素，变化的是type值为class 类组件

组件的属性 Props

类比于Vue中的props, 属性需要用小驼峰命名
Props会作为一个对象的属性，传递给函数组件的参数，传递给类组件的构造函数的参数

React中数据流动是自顶向下的
单向数据流：组件无法改变Props属性，数据属于谁谁才有权力改动。
(虽然可以改变props内对象的属性，react没有监控，但是这样很危险)

组件状态

类比于Vue的data
组件状态(state)：组件可以自行维护的数据

初始化状态：

  1. 在constructor中初始化

  2. js next，将state写在class内constructor外

状态变化：
不像vue对状态进行getter和setter，react必须使用this.setState({})
父类里提供了setState函数，重新设置状态，触发组件自动重新渲染

事件

在React中，事件就是组件的属性，需要用小驼峰命名。

向下传递事件，如果没有特殊处理，在事件处理函数中，this指向undefined

  1. 使用bind函数绑定this

    1. 在构造函数中this.handleClick = this.handleClick.bind(this)

    2. 传递时bind一个新的函数

  2. 使用箭头函数

    1. js next，class中handleClick = () => {}，这样写handleClick不在原型上，而在对象上。

    2. render函数中传递时使用箭头函数

深入认识setState

setState对状态的改变，可能是异步的。（类比于Vue的nextTick）

如果改变状态的代码处于某个HTML的事件中，则是异步的，否则是同步的。

如果希望改变状态后，做一些事情

  • 用setState的第二个参数-回调函数。
  • setState的函数

最佳实践

  1. 把所有的setState当做异步
  2. 不要信任setSate调用之后的状态
  3. 如果要使用改变之后的状态，使用回调函数（setState的第二个参数）
  4. 如果新的状态要根据之前的状态进行运算，使用函数的方式改变状态（setState的第一个参数用函数）

React的优化
React会对异步的setState进行优化，将队列中多次的setState调用完成后，然后对state进行改变，然后触发render。

生命周期

组件从诞生到销毁会经历一系列的过程。React在组件的生命周期中提供了一系列钩子函数，这些钩子函数会在适当的时候运行。

生命周期仅存在与类组件中，函数组件每次调用都是重新运行函数，旧的组件即刻被销毁。

旧版生命周期

React < 16.0.0

初始化阶段

  1. constructor
    同一个组件对象只会创建一次
    不能在组件第一次挂载之前使用setState。为了避免问题，不要在constructor中使用setState。

挂载阶段

  1. componentWillMount
    正常只会运行一次。（新版无）

  2. render
    渲染，挂载到虚拟DOM（JSX语法创建的React元素对象），最终会被渲染到真实DOM中。

  3. componentDidMount
    虚拟DOM已挂载到页面真实DOM后。
    通常把网络请求、启动计时器等一开始需要进行的操作，写到该函数中。

更新阶段（props或state变化）

  1. componentWillReceiveProps
    组件接收到新的属性值（新版无）

  2. shouldComponentUpdate
    组件是否该更新。(nextProps, nextState)
    指示React是否要重新render该组件，通过返回true/false来指定（默认返回true）

  3. componentWillUpdate
    组件即将被重新渲染

  4. render

  5. componentDidUpdate
    组件已完成渲染，往往在该函数中使用DOM操作

销毁阶段

  1. componentWillUnmount
    组件从DOM树中移除，通常在该函数中销毁一些组件依赖的资源，如计时器。

新版生命周期

变化主要是在挂载阶段和更新阶段。把几个will干掉了。

掉坑：组件的数据同时依赖 props和state，用componentWillReceiveProps来对变化的props进行setState。

React认为，某个数据的来源必须是单一的。

  1. static getDerivedStateFromProps
    用static getDerivedStateFromProps代替了原先的componentWillReceiveProps。属性和状态变化都会触发，static无法拿到this。

  2. getSnapshotBeforeUpdate
    dom构建完成，还未实际渲染到页面。
    通常用于实习一些附加的dom操作。
    该函数返回值，会作为componentDidUpdate的第三个参数。

传递元素内容

类比于Vue的插槽。
在自定义组件中传递内容，则React会将内容作为children属性传递
