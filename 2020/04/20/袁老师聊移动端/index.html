移动端开发

web app

编写的应用是运行在浏览器中，rem布局，H5开发、PWA

  • Html5：新的元素、语义化、新的web api（本地存储、多线程、socket、通知、地理位置）
  • H5开发：…民间搞的词，有可能指微信小程序

优势：开发成本低，易于更新
劣势：没有直接的入口，需要浏览器、效率不高

native app

编写的是原生应用，直接调用本地操作系统的系统

  • Android：java
  • IOS：swift、object-c
  • RN：把js代码编译成本地代码，最终运行的是本地代码
  • flutter：把dart代码编译成本地代码

优势：效率高、有直接的入口
劣势：开发成本高、升级麻烦

hybrid app

在移动app中，内置一个浏览器，运行的是js代码，把js解析成为本地代码

  • weex
  • cordava
  • apicloud
  • dcloud

学习记不住？

人的记忆是跟情感有关系的。
如果问你两周前那普通的一天，午餐吃了什么？那肯定记不住了。
但是问，两周前你和很久没见非常要好朋友吃了午餐，你肯定记得吃了什么，还会记得在哪一桌、怎么点菜的、吃了什么。我现在还记得17年高考完，我和pys去看神奇女侠，吃的硬皮披萨。

为什么练习这么重要？
因为练习会产生情感波动。

  • 做不出来、遇到问题，会觉得痛苦（所以痛苦，不一定坏处）
  • 完成之后的成就感

理解不了？

理解力的缺乏 -> 多去理解，反复理解（思考、思考一切）
要力量只能去练力量，要速度就去锻炼速度

没有动力？

经过一段时间进入了疲软期，感觉后面要做的东西好多。
知道自己是怎么回事，比对知道技术是怎么回事，更加重要。
动力的来源：

  1. 为了钱
    钱的动力不足，因为很多人对赚钱为目标也是有前提条件的：

    1. 安全
    2. 舒服、不能累

  2. 价值感
    被社会、被他人所肯定、所需要

  3. 好奇心

  4. 恐惧感
    不能作为长期动力。容易变成实用主义者，陷入初级工程师陷阱。靠ddl活着。

如何面对困难？

所有的困难背后都是机遇，所有的机遇都有困难。
往往很强的boss意味着通关之后的收益会很高。

很多人面对的不是困难，是内心的恐慌。
如果长期没有出现困难，要非常警惕。要么没有学了，要么是自己认为自己学懂了。永远不去打怪，永远获得不了经验，等级就没法提升。
