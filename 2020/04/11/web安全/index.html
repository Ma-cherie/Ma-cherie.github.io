web安全

发表于 2020-04-11

看别人面经时，发现经常问web安全的问题。一开始我看了一些文章还挺懵，后来学着学着突然开窍了一点。我觉得现在我能达到“了解一点点”的水平了。兴奋 (p≧w≦q)

XSS

XSS (Cross Site Script) 跨站脚本攻击，分为存储型和反射型

存储型XSS

对于有XSS漏洞的站点，攻击者通过类似于文章、评论的表单输入，将攻击代码存储到服务器。别的访问者访问页面的时候，这些恶意代码会执行，从而可以获得访问者的敏感信息等。

反射型XSS

攻击者通过在正常url上附加恶意代码，诱使用户点击，用户点击了url后跳转页面，恶意代码也会执行。

防御

对用户的输入进行转义，把尖角号转义成实体字符

CSRF

CSRF (Cross Site Request Forgery) 跨站请求伪造，利用用户的合法身份，发送恶意的请求。
比如用户打开了银行的网站，并且登录了，银行通过cookie验证用户身份。用户再打开攻击者的网站，这个恶意网站向银行网站发送请求。浏览器根据请求的url自动带上用户的cookie，这样攻击者就能跳过服务端验证，进行转账的等操作。

防御

  1. 检测Http首部的Refer字段，可以看到请求是从哪个站点发出的
  2. 重要操作要求输入验证码，确保是用户的行为
  3. CSRF token
    服务端生成随机的token，存在服务端，并放在页面的head标签内或者表单
    客户端发请求时，要携带这个token
    服务端通过拦截器验证token，验证请求的有效性
    (在CSRF中，攻击者是获取不到cookie的。因此也可以不存放在服务端，可以存放在cookie，但与此同时请求也要带上token，服务端拿到cookie里的token和请求中的token比对，如果相同则通过验证。)
